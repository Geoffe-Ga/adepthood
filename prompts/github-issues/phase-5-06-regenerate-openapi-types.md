# phase-5-06: Regenerate OpenAPI types and align frontend type definitions

**Labels:** `phase-5`, `full-stack`, `types`, `priority-high`
**Epic:** Phase 5 — Test Coverage & Security Hardening
**Estimated LoC:** ~200

## Problem

`frontend/src/api/types.ts` was auto-generated by openapi-typescript but is
completely stale. It only contains schemas for 4 routes (`/`, `/v1/energy/plan`,
`/practice_sessions/`, `/practice_sessions/{user_id}/week_count`) while the
backend now has 20+ routes across 14 router modules. The frontend has worked
around this by defining all types manually in `api/index.ts` (lines 126–618),
creating a dual-type system that will inevitably drift from the backend.

Current state: `types.ts` still references `PracticeSession` with `practice_id`
and `stage_number` fields that no longer match the actual schema (the backend
now uses `user_practice_id` on practice sessions).

## Scope

Regenerate `types.ts` from the live backend OpenAPI schema and migrate
`api/index.ts` to use generated types where possible. Manual types remain only
for types not represented in the OpenAPI spec.

## Tasks

1. **Generate fresh OpenAPI schema**
   - Run the backend locally to export `/openapi.json`
   - Run `npx openapi-typescript` against the exported schema
   - Replace `frontend/src/api/types.ts` with the output

2. **Align `api/index.ts` with generated types**
   - Replace manually defined interfaces (`ApiHabit`, `ApiGoal`, `JournalMessage`,
     `Stage`, `ContentItem`, etc.) with re-exports from `types.ts` where the
     generated type matches
   - Keep manual types only for shapes not in the OpenAPI spec (e.g., local
     transformations like `toLocalHabit`)
   - Update any generic `request<T>` calls to use generated response types

3. **Add a script to regenerate types**
   - Add `"generate:types"` script to `frontend/package.json`
   - Document the regeneration workflow in a comment at the top of `types.ts`

## Acceptance Criteria

- `types.ts` contains schemas for all backend endpoints
- `api/index.ts` imports from `types.ts` instead of duplicating interfaces
- `npx tsc --noEmit` passes with no type errors
- Frontend lint passes
- No runtime behavior changes
